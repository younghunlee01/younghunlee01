## 프로젝트 목표

AWS 클라우드 환경에서 3-Tier 웹 서비스 아키텍처를 직접 설계하고 구축한다. 각 계층을 독립적으로 분리하여 확장성, 가용성, 보안을 확보하는 것을 목표로 한다.

### 1. 3-Tier 아키텍처 구현
- Nginx(Web), Tomcat(WAS), RDS(DB)를 독립된 계층으로 구성
- 계층 상호 간의 데이터 연동

### 2. 서비스 부하 분산
- ALB(Application Load Balancer)를 사용하여 트래픽을 분산
- 특정 서버의 장애가 서비스 전체로 확산되지 않도록 구성

### 3. 네트워크 영역 분리
- Public과 Private 서브넷을 용도별로 분리
- 보안 그룹(SG)과 Bastion Host를 활용해 외부 노출 최소화

## 인프라 전환: 온프레미스 → AWS

기존 데이터 센터(온프레미스) 환경의 구성 요소를 AWS 관리형 서비스로 대응·전환하였다.

### 데이터 센터 연결
| 온프레미스 | AWS |
|-----------|-----|
| 라우터/방화벽으로 외부망 연동 | VPC의 라우터, 방화벽 이용 |
| 전용선, VPN을 이용한 연결 | 전용선, VPN 포함 연결 옵션 제공 |

### 네트워크 보안
| 온프레미스 | AWS |
|-----------|-----|
| 방화벽, IP 기반 접근 통제(ACL) | 방화벽 정책 — 보안 그룹 |
| 웹방화벽(WAF), DDoS 방어 등 | IP 기반 접근 통제(ACL) — NACL / Managed WAF, DDoS 방어 제공 |

### 확장성 / 가용성
| 온프레미스 | AWS |
|-----------|-----|
| L4/L7 부하 분산(LB) | 자동 확장 기반 부하 분산 서비스 |
| 개별 장비 업그레이드 | 고가용성을 위한 가용 영역(AZ) |
| 재해복구 센터(DR) 추가 연결 | — |
